---
title: Mattermost
description: Send task notifications from Kaneo to a Mattermost channel using incoming webhooks.
---

Kaneo can post task activity to a Mattermost channel whenever events occur in a project. This uses Mattermost's incoming webhook feature — no bot account or OAuth flow required.

## Create a Mattermost webhook

1. In Mattermost, go to **Main Menu → Integrations → Incoming Webhooks**.
2. Click **Add Incoming Webhook**.
3. Give it a title (e.g. "Kaneo") and select the default channel where notifications should be posted.
4. Click **Save**, then copy the webhook URL. It will look like:

```
https://your-mattermost-server.com/hooks/xxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

<Warning>
If the Integrations menu is not visible, your Mattermost admin may need to enable incoming webhooks under **System Console → Integrations → Integration Management**.
</Warning>

## Connect Mattermost in Kaneo

1. Open **Settings** in the sidebar.
2. Under **Projects**, select the project you want to connect.
3. Open the **Integrations** tab.
4. Expand the **Mattermost** section.
5. Paste your webhook URL.
6. Optionally add a channel name label for your own reference.
7. Choose which events to subscribe to (see below).
8. Click **Connect**.

Once connected you can enable or disable the integration with the toggle, update event subscriptions, or disconnect entirely.

## Supported events

| Event | Default | Description |
| --- | --- | --- |
| Task created | On | A new task is created in the project |
| Task status changed | On | A task moves to a different status column |
| Task priority changed | Off | A task's priority is changed |
| Task title changed | Off | A task's title is edited |
| Task description changed | Off | A task's description is edited |
| Task comment created | On | A new comment is added to a task |

You can toggle any combination of these events in the integration settings at any time.

## Message format

Kaneo sends messages using Mattermost's legacy attachment format with:

- A clickable title linking to the task in Kaneo
- A description of the event (e.g. "Task moved from In Progress to Done")
- Fields for project, status, priority, and the user who triggered the event
- An indigo sidebar colour for easy visual identification

Make sure [`KANEO_CLIENT_URL`](/core/installation/environment-variables) is set to your public Kaneo URL so task links work correctly.

## Updating the webhook

To change the webhook URL after connecting, paste the new URL in the webhook field and click **Save Changes**. If you leave the field empty, the existing URL is kept.

## Disconnecting

Click **Disconnect** to remove the integration. This deletes the stored webhook URL and all event settings for that project. You can reconnect at any time by pasting a new webhook URL.
